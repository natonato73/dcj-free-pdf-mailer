# DCJ Free PDF Mailer v1.3.0 開発計画メモ

## 目的

v1.3.0では、販売前の完成度を高めるため、管理画面の使いやすさと運用機能を段階的に改善しました。

現在は v1.3.0 の正式リリース準備中です。タグ作成、GitHub Release作成、zip作成はこのメモの時点ではまだ行っていません。

## 優先順位

### 優先1：管理画面UI整理 第2弾

状態：実装済み

目的：

- 管理画面を見やすくする
- PDF設定、送信ログ、購読者リスト、メール診断の区切りを分かりやすくする
- ボタンや説明文を整理する
- 販売・納品後に初心者でも迷いにくい画面にする

候補：

- PDF設定一覧の見やすさ改善
- 新規追加フォームの説明整理
- 確認用フォームの表示整理
- 送信ログ・購読者リスト周辺の説明文整理
- CSVボタンの配置整理
- 注意文の重複削減

### 優先2：購読者の手動追加

状態：実装済み

目的：

- 管理画面からメールアドレスを1件ずつ追加できるようにする
- テスト用購読者や、同意済みの少数顧客を手動登録できるようにする

実装済み内容：

- 購読者リスト内に **購読者を手動追加** フォームを追加
- メールアドレス、言語、登録元メモ、同意確認済みチェックを入力
- メールアドレス形式を確認
- 言語は `ja` または `en` を必須にし、言語未指定の新規登録は許可しない
- 同意確認済みチェックがない場合は追加しない
- 既に登録済み、または配信停止済みのメールアドレスは追加しない
- 配信停止済みアドレスを自動で購読中に戻さない
- 手動追加した購読者は、既存の購読者リスト表示とCSV出力に同じ形式で反映
- 手動追加フォーム付近にも成功・エラーメッセージを表示

注意点：

- 同意確認が必要
- 登録元が手動追加であることを記録する
- 重複メールアドレス対策が必要
- 配信停止済みアドレスの扱いを慎重に決める

### 優先3：購読者CSVインポート

状態：実装済み

目的：

- 既存の同意済みリストをCSVで取り込めるようにする

実装済み内容：

- 購読者リスト内に **購読者CSVインポート** フォームを追加
- CSVファイル選択、2つの確認チェック、プレビュー、インポート実行に対応
- 1行目をヘッダーとして扱い、必須列 `email`、`lang`、任意列 `source_note`、`consent_confirmed` を利用
- `lang` は `ja` または `en` を必須にし、言語未指定の新規登録は許可しない
- `consent_confirmed` は `yes` / `1` / `true` を同意済みとして扱う
- `consent_confirmed` 列がある場合は、同意済み値の行だけ登録候補にする
- プレビューで読み込み件数、登録予定件数、既存登録済みスキップ件数、CSV内重複スキップ件数、入力不正件数、同意未確認スキップ件数、エラー一覧を表示
- 既に登録済み、または配信停止済みのメールアドレスは追加しない
- 配信停止済みアドレスを自動で購読中に戻さない
- CSV内重複、不正メール、言語未指定、言語不正、同意未確認の行はスキップ
- CSVインポートした購読者は、既存の購読者リスト表示とCSV出力に同じ形式で反映
- DB構造、既存CSV出力処理、配信停止URL処理は変更しない
- 既存データの言語未指定は後方互換として残しつつ、手動追加とCSVインポートでは新しい言語未指定データを増やさない

注意点：

- 誤登録リスクが高い
- CSV形式チェックが必要
- 重複排除が必要
- 同意済みリストだけを対象にする注意文が必要

### 後回し：外部メール配信サービスAPI連携

理由：

- APIキー管理が必要
- 配信停止同期が必要
- 外部サービス依存が増える
- サポート負担が大きくなる

現時点では、メール配信用CSVによる手動運用を優先します。

## v1.3.0で最初に着手する内容

まずは、管理画面UI整理 第2弾から開始しました。購読者の手動追加と購読者CSVインポートも実装済みです。

初回の実装候補：

1. 管理画面の主要セクションに説明を追加または整理
2. PDF設定一覧の上部説明を整理
3. 送信ログ・購読者リストの説明を短く分かりやすくする
4. CSVボタン周辺の表示を少し見やすくする
5. 変更は小さく分けて実装し、LocalWPで確認する

追加で実装済み：

- 購読者の手動追加
- 手動追加フォーム付近の成功・エラー表示
- 購読者CSVインポート
- CSVインポートのプレビュー、実行、成功・スキップ・エラー件数表示

## 開発ルール

- masterを直接触らない
- 作業ブランチを作る
- .codex は触らない
- git add . は使わない
- 変更対象ファイルを明示して git add する
- PHP変更後は php -l dcj-free-pdf-mailer.php を実行する
- LocalWPで管理画面表示を確認する
- PRでmasterへ反映する
- マージ後はローカルmasterを同期し、作業ブランチを削除する
