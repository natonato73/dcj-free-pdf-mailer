# DCJ Free PDF Mailer v1.3.0 開発計画メモ

## 目的

v1.3.0では、販売前の完成度を高めるため、管理画面の使いやすさと運用機能を段階的に改善します。

商品化はまだ急がず、まずは実運用・納品・保守時に迷いにくい状態を目指します。

## 優先順位

### 優先1：管理画面UI整理 第2弾

目的：

- 管理画面を見やすくする
- PDF設定、送信ログ、購読者リスト、メール診断の区切りを分かりやすくする
- ボタンや説明文を整理する
- 販売・納品後に初心者でも迷いにくい画面にする

候補：

- PDF設定一覧の見やすさ改善
- 新規追加フォームの説明整理
- 確認用フォームの表示整理
- 送信ログ・購読者リスト周辺の説明文整理
- CSVボタンの配置整理
- 注意文の重複削減

### 優先2：購読者の手動追加

目的：

- 管理画面からメールアドレスを1件ずつ追加できるようにする
- テスト用購読者や、同意済みの少数顧客を手動登録できるようにする

注意点：

- 同意確認が必要
- 登録元が手動追加であることを記録する
- 重複メールアドレス対策が必要
- 配信停止済みアドレスの扱いを慎重に決める

### 優先3：購読者CSVインポート

目的：

- 既存の同意済みリストをCSVで取り込めるようにする

注意点：

- 誤登録リスクが高い
- CSV形式チェックが必要
- 重複排除が必要
- 同意済みリストだけを対象にする注意文が必要

### 後回し：外部メール配信サービスAPI連携

理由：

- APIキー管理が必要
- 配信停止同期が必要
- 外部サービス依存が増える
- サポート負担が大きくなる

現時点では、メール配信用CSVによる手動運用を優先します。

## v1.3.0で最初に着手する内容

まずは、管理画面UI整理 第2弾から開始します。

初回の実装候補：

1. 管理画面の主要セクションに説明を追加または整理
2. PDF設定一覧の上部説明を整理
3. 送信ログ・購読者リストの説明を短く分かりやすくする
4. CSVボタン周辺の表示を少し見やすくする
5. 変更は小さく分けて実装し、LocalWPで確認する

## 開発ルール

- masterを直接触らない
- 作業ブランチを作る
- .codex は触らない
- git add . は使わない
- 変更対象ファイルを明示して git add する
- PHP変更後は php -l dcj-free-pdf-mailer.php を実行する
- LocalWPで管理画面表示を確認する
- PRでmasterへ反映する
- マージ後はローカルmasterを同期し、作業ブランチを削除する
