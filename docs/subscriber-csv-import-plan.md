# DCJ Free PDF Mailer 購読者CSVインポート仕様メモ

## 目的

v1.3.0の次期開発項目として、同意確認済みの購読者メールアドレスをCSVから取り込み、既存の購読者リストへ登録できるようにするための仕様を整理します。

このメモは仕様整理用です。現時点ではCSVインポート機能を実装せず、プラグイン本体のPHPも変更しません。

## 前提

- 最新リリースは v1.2.2
- 現在は v1.3.0 開発中
- 購読者の手動追加機能は実装済み
- 既存の購読者リスト保存形式を使う
- DB構造は変更しない
- 既存の購読者リストCSV出力、メール配信用CSV出力は壊さない
- メール配信サービスやCRMの完全代替ではなく、同意済みリストの取り込み補助として扱う

## 1. 機能概要

管理画面からCSVファイルを選択し、同意確認済みの購読者メールアドレスを購読者リストへ一括登録できるようにする機能です。

取り込み先は既存の `dcj_fpm_subscribers` の保存形式とし、手動追加やフォーム経由で登録された購読者と同じ一覧・検索・CSV出力で扱える状態を目指します。

## 2. 想定用途

- 既に同意取得済みの少量から中量のメールアドレスを移行する
- 別のフォームや手作業で管理していた同意済みリストを登録する
- 納品時にクライアントから受け取った同意済みリストを初期登録する
- テスト環境や本番移行時に、確認済みの購読者リストをまとめて投入する

大量配信基盤、CRM、外部メール配信サービスのリスト管理を完全に置き換える用途は想定しません。

## 3. インポート対象CSV形式

1行目をヘッダー行として扱うCSVを基本形式にします。

想定ヘッダー例：

```csv
email,lang,source_note,consent_confirmed
test@example.com,ja,manual-import,yes
```

区切り文字はカンマを基本とし、ExcelやGoogleスプレッドシートから出力した一般的なCSVを取り込めることを目標にします。

## 4. 必須列

必須列は以下とします。

- `email`

`email` は購読者メールアドレスです。空欄、形式不正、複数メールアドレスの混在はエラーにします。

## 5. 任意列

任意列の候補は以下とします。

- `lang`
- `source_note`
- `consent_confirmed`

`lang` は `ja` または `en` を想定します。空欄の場合は既存の手動追加仕様に合わせて既定値を決めます。

`source_note` は登録元メモとして使います。既存保存形式に直接対応する項目がない場合は、`source_title` など既存フィールドに収まる範囲で扱い、DB構造は変更しません。

`consent_confirmed` は同意確認済みであることをCSV上でも明示するための候補列です。値は `yes` / `1` / `true` などを許可する案を検討します。

## 6. 入力値チェック

インポート前に以下を確認します。

- CSVファイルが選択されていること
- ファイル拡張子またはMIMEタイプがCSVとして妥当であること
- ヘッダー行が存在すること
- 必須列 `email` が存在すること
- メールアドレス形式が妥当であること
- `lang` が許可値の範囲内であること
- 1行内に複数メールアドレスが入っていないこと
- 空行を無視できること
- 極端に長い値を受け付けないこと
- 管理画面のnonceと権限チェックを通過していること

不正行がある場合でも、全体を止めるか、正常行だけ取り込むかはプレビュー時に分かるようにします。

## 7. 同意確認の扱い

CSVインポートは、同意確認済みのメールアドレスだけを取り込む前提にします。

実装時は以下の両方を検討します。

- 画面側に「このCSVのメールアドレスは全て同意確認済みです」という実行前チェックボックスを置く
- CSVに `consent_confirmed` 列を持たせ、行ごとに同意確認済みか確認できるようにする

安全側に倒す場合、実行前チェックボックスは必須とし、`consent_confirmed` 列が存在する場合は同意済み値の行だけを取り込みます。

`consent_confirmed` 列がないCSVを許可するかどうかは、実装時に管理画面の説明と合わせて決めます。

## 8. 重複メールアドレスの扱い

既に購読者リストへ登録済みのメールアドレスは上書きしません。

- 既存データの `lang`
- 登録元情報
- 登録日時
- 最終確認日時
- ステータス

これらはCSVインポートによって変更しません。

CSV内に同じメールアドレスが複数ある場合は、最初の1件だけを候補とし、2件目以降は重複としてスキップまたはエラー一覧へ表示します。

## 9. 配信停止済みメールアドレスの扱い

配信停止済みメールアドレスは、自動で購読中に戻しません。

既存データに `status = unsubscribed` のメールアドレスがある場合、CSVに含まれていても取り込み対象外にします。

再購読が必要な場合は、本人による再同意フローまたは管理者が明確に判断できる別操作で扱い、CSVインポートの副作用として復帰させない方針にします。

## 10. 文字コードの扱い

基本はUTF-8のCSVを想定します。

日本語環境での運用を考慮し、将来的にはShift_JISまたはWindows-31JのCSVも検討対象にします。ただし、v1.3.0で対応する場合も既存CSV出力処理に影響しない形にします。

BOM付きUTF-8は、ヘッダー名の判定で問題が出ないように扱う必要があります。

## 11. 最大件数の目安

WordPressオプションに保存する既存形式を使うため、大量件数の取り込みは想定しません。

v1.3.0では、目安として1回あたり100件から500件程度までを安全な範囲として検討します。

上限を超えるCSVは、プレビュー時点で警告または取り込み不可にします。大量リスト運用が必要な場合は、外部メール配信サービスや専用DB設計を検討対象にします。

## 12. インポート前プレビューの必要性

インポート前プレビューは必要です。

プレビューでは以下を表示します。

- 読み込み行数
- 登録予定件数
- 既存登録済みのためスキップされる件数
- 配信停止済みのためスキップされる件数
- メールアドレス形式不正などのエラー件数
- 同意確認列が不足または未確認の件数

実登録前に、管理者が取り込み内容を確認できる状態にします。

## 13. 実行前確認チェックボックス

実行前に以下のような確認チェックボックスを必須にします。

- このCSVに含まれるメールアドレスは、メール配信またはお知らせ受信について同意確認済みです。
- 配信停止済みのメールアドレスを自動で購読中に戻さないことを理解しました。

チェックがない場合はインポートを実行しません。

## 14. 成功・失敗件数の表示

インポート後は、管理画面に結果を表示します。

- 登録成功件数
- 既存登録済みスキップ件数
- 配信停止済みスキップ件数
- CSV内重複スキップ件数
- 入力不正件数
- 同意未確認スキップ件数

成功と失敗が混在した場合も、どの行がどう扱われたか分かるようにします。

## 15. エラーCSVまたはエラー一覧の考え方

v1.3.0では、まず画面上のエラー一覧表示を優先します。

表示候補は以下です。

- 行番号
- メールアドレス
- エラー理由
- スキップ理由

件数が多い場合に備え、将来的にはエラーCSVのダウンロードも検討します。ただし、初期実装では画面表示だけでもよいものとします。

## 16. CSVインポートで変更しないこと

CSVインポートでは以下を変更しません。

- DB構造
- 既存の購読者リスト保存形式
- 既存の購読者CSV出力処理
- 既存のメール配信用CSV出力処理
- 既存購読者の登録情報
- 配信停止済み購読者のステータス
- PDF設定
- 送信ログ
- メール本文テンプレート
- 外部メール配信サービス連携

## 17. 実装時の注意点

- 管理者権限とnonceを必ず確認する
- CSVアップロード処理で一時ファイルを安全に扱う
- メールアドレスは小文字化して比較する
- 既存登録済み判定と配信停止済み判定を先に行う
- 既存データを上書きしない
- `status` は新規登録時のみ `subscribed` を設定する
- `optin_datetime` と `last_seen_datetime` はインポート実行時刻を使う案を検討する
- 登録元はCSVインポートであることが分かる値にする
- 既存の手動追加処理と重複しすぎる場合は、共通ヘルパー化を検討する
- ただし、v1.3.0では大きなリファクタリングを避ける
- CSV出力クラスや既存エクスポート処理に副作用を出さない

## 18. v1.3.0で実装する範囲

v1.3.0で実装する場合の最小範囲は以下です。

- 管理画面からCSVを選択
- ヘッダー付きCSVを読み込み
- `email` 必須列を確認
- 任意で `lang`、`source_note`、`consent_confirmed` を扱う
- インポート前プレビューを表示
- 実行前確認チェックボックスを必須にする
- 同意確認済みの行だけ登録する
- 既存登録済みメールアドレスは上書きしない
- 配信停止済みメールアドレスは購読中に戻さない
- 成功件数、スキップ件数、エラー件数を表示する
- エラー一覧を画面に表示する
- 既存の購読者リスト表示とCSV出力に反映する

## 19. 後回しにする範囲

以下はv1.3.0では後回しにします。

- 大量件数向けの専用DBテーブル
- バックグラウンド処理
- 分割インポート
- インポート履歴の永続保存
- エラーCSVダウンロード
- 外部メール配信サービスAPI連携
- CRM連携
- 自動文字コード判定の高度化
- 列マッピングUI
- 既存購読者情報の一括更新
- 配信停止済みメールアドレスの一括再購読
